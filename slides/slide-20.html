Comparativa técnica de mitigación de vulnerabilidades SQL Injection.

cancel

ANTES

Vulnerable
code 
String query = "SELECT * FROM users WHERE name = '" 
               + userName + "'";
warning

Riesgo Crítico: La concatenación directa de strings permite la inyección de comandos SQL maliciosos a través de la variable userName.

check_circle

DESPUÉS

Seguro
code 
PreparedStatement stmt = con.prepareStatement(
    "SELECT * FROM users WHERE name = ?");
stmt.setString(1, userName);
verified_user

Solución: El uso de Prepared Statements separa la lógica SQL de los datos, sanitizando automáticamente la entrada y neutralizando ataques.
