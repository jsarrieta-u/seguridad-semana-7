Paso 2: Análisis SAST

Detección de SQL Injection en código fuente

search Auditoría de Código

La herramienta de análisis estático (SonarQube) detecta una vulnerabilidad crítica: Inyección SQL. El código utiliza concatenación directa de cadenas para construir la consulta, permitiendo que un atacante manipule la entrada para ejecutar comandos arbitrarios en la base de datos.

bug_report SonarQube: SQL Injection vulnerability (CWE-89) - Line 12

check_circle UserController.java (Fixed)
SECURE
10 // Authenticate user using PreparedStatement
11 public String getUser(String userId) {
12 String query = "SELECT * FROM users WHERE id = ?";
13 PreparedStatement pstmt = connection.prepareStatement(query);
14 pstmt.setString(1, userId);
15 ResultSet rs = pstmt.executeQuery();
16 return rs.toString();
17 }

verified Solución: Uso de PreparedStatement con parámetros
